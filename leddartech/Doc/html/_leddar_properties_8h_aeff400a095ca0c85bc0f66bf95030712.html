enum LdProperties

List all the Ids recognized by the LeddarGetProperty() and LeddarSetProperty() functions. Note that not all ids are supported by all devices and some are read-only.

PID_OVERSAMPLING_EXPONENT 

The power of 2 of the number of oversamplings to perform between base samples.

This property is available for reading and writing. Oversampling can only take values that are a power of two. This property can have a value between 0 and 3, giving oversampling of 1, 2, 4 and 8.

PID_OVERSAMPLING 

The number of oversamplings to perform between base samples.

This property is only available for reading. To change the value use the PID_OVERSAMPLING_EXPONENT property. Possible values are: 1, 2, 4, and 8.

PID_ACCUMULATION_EXPONENT 

The power of 2 of the number of accumulations to perform.

This property is available for reading and writing. The number of accumulations can only take values that are a power of two. This property can have a value between 0 and 10 giving a number of accumulations of 1, 2, 4, 8, 16, 32, 64, 128, 256, 512 or 1024.

Note
Some older version of Leddar™ firmware, have a maximum value of 8 for this property.
PID_ACCUMULATION 

The number of accumulations to perform.

This property is only available for reading. To change the value use the PID_ACCUMULATION_EXPONENT property. Possible values are: 1, 2, 4, 8, 16, 32, 64, 128, 256, 512 and 1024.

PID_BASE_POINT_COUNT 

The number of base samples acquired by the sensor.

The number of samples that the sensor acquires can be changed. However the number of samples set here is the "base" number before oversampling. So for example setting this property to 30 and oversampling to 8 will give 240 samples.

PID_THRESHOLD_OFFSET 

An offset to increase the detection threshold.

By default this property is 0 which makes the sensor very sensitive. However depending on your application this may yield many "false" detections with low amplitude that are just noise. To eliminate them you can increase this value. The value to use is application dependant and must be found by experimentation.

PID_ZONE_NEAR_LIMIT 

Defines the near limit of a segment of a zone in meters.

This property is available for reading and writing. The index specifies the segment and zone with this formula: Index = Zone*16 + Segment. The value cannot be negative.

Note
This property is only supported by IS16 industrial Leddar™ sensor.
PID_ZONE_FAR_LIMIT 

Defines the far limit of a segment of a zone in meters.

This property is available for reading and writing. The index specifies the segment and zone with this formula: Index = Zone*16 + Segment. The value cannot be negative.

Note
This property is only supported by IS16 industrial Leddar™ sensor.
PID_ZONE_ENABLED 

The "enabled" state of a zone.

This property is available for reading and writing. The index specifies the zone. When a zone is disabled, it is not processed, whatever the "enabled" state of its individual segments. The zone will be enabled by any value other than 0.

Note
This property is only supported by IS16 industrial Leddar™ sensor.
PID_ZONE_SEGMENT_ENABLED 

The "enabled" state of a segment of a zone.

This property is available for reading and writing. The index specifies the segment and zone with this formula: Index = Zone*16 + Segment. The segment will be enabled by any value other than 0.

Note
This property is only supported by IS16 industrial Leddar™ sensor.
PID_ZONE_RISING_DEBOUNCE 

Control of the debouncing done on a rising edge.

This property is available for reading and writing. The index specifies the zone. The value must be between 0 and 65535 with 0 equivalent to 1. The value is the number of consecutive acquisitions with a detection in the zone before the output switches to the "detected" state.

Note
This property is only supported by IS16 industrial Leddar™ sensor.
PID_ZONE_FALLING_DEBOUNCE 

Control of the debouncing done on a falling edge.

This property is available for reading and writing. The index specifies the zone. The value must be between 0 and 65535 with 0 equivalent to 1. The value is the number of consecutive acquisitions without a detection in the zone before the output switches to the "not detected" state.

Note
This property is only supported by IS16 industrial Leddar™ sensor.
PID_OUTPUT_PNP 

The type of output for each zone.

This property is available for reading and writing. The index specifies the zone. A value of 0 sets the output in NPN mode, any other value sets it in PNP mode.

Note
This property is only supported by IS16 industrial Leddar™ sensor.
PID_OUTPUT_INVERTED 

Control of the output inversion for each zone.

This property is available for reading and writing. The index specifies the zone. For a value of 0 the output will be asserted when there is a detection, de-asserted otherwise. For any other value, the output will be de-asserted when there is a detection, asserted otherwise.

Note
This property is only supported by IS16 industrial Leddar™ sensor.
PID_TEACH_MARGIN 

Safety margin in meters for auto-teach limits.

This property is available for reading and writing. When an auto-teach is performed (CID_START_TEACH command), the far limit will be set to the current detection minus this margin.

Note
This property is only supported by IS16 industrial Leddar™ sensor.
PID_TEACH_STATE 

Used with CID_START_TEACH to know when the process is completed.

This property is only available for reading. The teaching process is asynchronous. When the CID_START_TEACH command is executed, the process is started. To know when the process is completed, use this property. If the auto-teach process is running, this property will be non-zero.

Note
This property is only supported by IS16 industrial Leddar™ sensor.
PID_AUTOMATIC_LED_INTENSITY 

Control of automatic LED intensity.

This property is available for reading and writing. A value of 0 will set manual intensity mode (as controlled by PID_LED_INTENSITY). Any other value will set automatic mode where the LED intensity is computed by the sensor.

PID_CHANGE_DELAY 

Number of measurements of filtering for automatic LED control.

This property is available for reading and writing. When PID_AUTOMATIC_LED_INTENSITY is true, this parameter controls the reactivity of the control. The higher this value, the slower the sensor will be to adjust the LED intensity to condition changes. This property is specified as a number of measurement, so the effective delay will depend on the current measurement rate.

PID_OBJECT_DEMERGING 

Control of object demerging (detection of 2 objects close to each other).

This property is available for reading and writing. A value of 0 will disable object demerging. Any other value will enable it.

Note
Whatever the value of this property, object demerging is not performed when the sensor is running at a measurement rate above 10 Hz.
PID_MEASUREMENT_RATE 

Simpler access to acquisition parameters.

This property is available for reading and writing. This property is an simplified way of setting acquisition parameters. On reading, it will return the measurement rate for the current parameters. On writing, the parameters will be set to achieve the measurement rate. However the value set must be one of the predefined rates or parameters will be left unchanged. To ensure you set the proper value, use one of the LD_MEASUREMENT_RATE constants defined in LeddarProperties.h.

Note
This property is only supported by IS16 industrial Leddar™ sensor.
PID_NAME 

The name given to the device.

This property is available for reading and writing.
