Leddar™ SDK .NET Interface

Introduction

The Leddar™ SDK includes a .NET object oriented interface. This allows easy integration in high-level environments that support .NET like MATLAB™ or LabVIEW™

Click any link to LeddarNET for the main entry point to the reference guide.

Usage

Two versions of the LeddarNET DLL are provided:

  • LeddarNET.dll compiled with VisualStudio 2012 and targeting .NET 4.0
  • LeddarNET2.dll compiled with VisualStudio 2008 and targeting .NET 2.0

Both files are installed in the main Leddar™ software installation directory. Use the version that is appropriate for your environment.

There are 3 examples showing how to use the LeddarNET interface:

  • LeddarNET Example
  • MATLAB™ Integration Example
  • LabVIEW™ Integration Example

By looking at the example that best fits your needs, you should be able to quickly understand how to use the interface. However here is a quick summary of the typical steps involved:

  • Create a LeddarNET.Device object.
  • Optionnally use the LeddarNET.Device.ListSensors function to get a list of available sensors.
  • Connect to the chosen sensor using LeddarNET.Device.Connect (alternatively you can use LeddarNET.Device.LoadRecord to open a record file).
  • Once connected you may read configuration settings using LeddarNET.Device.GetProperty and modify them using LeddarNET.Device.SetProperty.
  • If you modify the configuration you must commit the changes to the sensor using LeddarNET.Device.WriteConfiguration.
  • You can start to read data from the sensor using LeddarNET.Device.StartDataTransfer (see next paragraph for more details on reading data).
  • Once finished use LeddarNET.Device.Disconnect to close the connection to the sensor (or the record file).

Once data transfer is started, data is continuously read from the sensor (however when using a record file you must call LeddarNET.Device.StepForward or other movement functions to get data). To read data you must register a handler with the LeddarNET.Device.DataReceived event. All data should be read while in that handler to ensure coherency, using LeddarNET.Device.Detections or LeddarNET.Device.GetResult.
