MATLAB™ Integration Example

Introduction

MATLAB™ offers integration of the .NET framework. LeddarTech has built a simple user interface demonstrating the integration of the Leddar™ functionalities. It implements Leddar™ functions plus a bit of scripting to build the user interface.

Description and usage

To use the Leddar™ .NET classes in a MATLAB™ script:

  • Be familiar with the Matlab™ .NET package. It allows calling base .NET functions by prefixing the function names with NET..
  • Call NET.addAssembly using the path to your LeddarNET2.dll. This makes the LeddarNET assembly visible to MATLAB™.
  • Instantiate a Leddar™ device using the Device constructor: dev = LeddarNET.Device; . All Leddar™ methods are available via this object.
  • Don't forget to Dispose of it when your work is done. Unlike LabVIEW™, a new device is created each time you call the constructor. So destructing it is a good practice when you call your scripts repeatedly.

Leddar2Matlab Function

This function is the entry point of the example. It creates the user interface, links the UI objects to their callback function then exits. Start by entering the path to your recorded LTL file or the name of the USB device to connect to. As usual, you can obtain the list of connected USB devices using the main Leddar™ Configuration Software coming with the sensor. Then select the desired connection type before hitting the START button. The AUTO-FIT button automatically adjusts the view to zoom on the current detections. Release the button to stop auto-adjusting if the screen moves too fast. The RESET button sets the X and Y limits of the axes to the MATLAB™ default "auto" and thus zooms out to see everything inside. It won't have any visible effect while the AUTO-FIT is ON.

LeddarOpen Function

It is used to initiate communication between the MATLAB™ application and a device or a recorded sensor session. It takes as input the path to the file or the USB device address and returns the connection success state as well as the .NET object used to send and receive commands to the device. Also in LeddarOpen, we link the listener of the DataReceived event to the device. A DataReceived_CB function is created and its handle is passed to the MATLAB™ addlistener function. We use the DATA global variable to transfer the detection array between the callback and the LeddarGetData function. Be sure to not use this global variable name in your own code or it might interfere.

LeddarGetData Function

This function waits for new data to arrive until timeout and returns these detections as well as the new data status. When in recorded log mode, it allows to specify the frame index from which to get the data.

LeddarClose Function

This function terminates a Leddar™ connection and cleans up everything. The .NET device object is not usable any more after calling it. A new call to LeddarOpen is required.

Note
This example was written and / tested with MATLAB™ R2013a but should work in any recent version.
